# Slider UX Overhaul Plan

## Problem
~40+ sliders across the FFB Tuner UI are confusing to use: no way to type exact values, no defaults, no undo, all look the same, wide-range sliders are hard to fine-tune.

## Approach: Custom `LabeledSlider` UserControl

Create a reusable WPF `UserControl` that encapsulates all slider UX improvements. Each of the ~40 slider instances in `MainWindow.xaml` gets replaced by a single `<controls:LabeledSlider .../>` tag.

### Dependency Properties on `LabeledSlider`

| Property | Type | Purpose |
|----------|------|---------|
| `Value` | `double` | Two-way bound to ViewModel property |
| `Minimum` | `double` | Slider minimum |
| `Maximum` | `double` | Slider maximum |
| `TickFrequency` | `double` | Step size |
| `Label` | `string` | e.g. "Mz (Self-Align)" |
| `Description` | `string` | Tooltip/help text |
| `DefaultValue` | `double?` | Profile default for reset indicator |
| `RecommendedMin` | `double?` | Green zone lower bound |
| `RecommendedMax` | `double?` | Green zone upper bound |
| `StringFormat` | `string` | e.g. `"{0:F3}"` or `"{0:F0}"` |
| `IsLogarithmic` | `bool` | Use log scale mapping |
| `SectionColor` | `Brush` | Accent color for this section |
| `IsEnabled` | `bool` | Enable/disable with CheckBox binding |
| `ShowCheckBox` | `bool` | Show enable/disable checkbox |
| `IsChecked` | `bool` | CheckBox state (two-way) |
| `Suffix` | `string` | Unit label (Hz, deg, Nm, km/h) |
| `Width` of Label | `double` | Label width for alignment (default 160) |

### Visual Layout

```
[CheckBox] [Label                    ] [Slider═══════════════════] [Value TextBox] [Reset btn]
           [Description text below, dimmed, wraps                                           ]
```

## Feature Details

### 1. Editable Value TextBox
- The numeric display is a `TextBox` (styled to look like current read-only display)
- Click to focus → type new value → Enter to apply → Escape to revert
- Validation: clamp to Min/Max range, show red border if invalid input
-LostFocus also applies the value
- Only enters edit mode on click/focus, not on slider drag

### 2. Right-Click Context Menu
On the slider or the entire control:
- **Reset to Default** — restores `DefaultValue`
- **Copy Value** — copies current value to clipboard
- **Paste Value** — pastes from clipboard (validates numeric)
- Separator
- **Set as Default** — updates the stored default to current value

### 3. Reset-to-Default Button
- Small icon button (↺ or ×) to the right of the value
- Only visible when value differs from default (or always visible, dimmed when at default)
- Clicking it restores the `DefaultValue`
- ToolTip: "Reset to default (X.XXX)"

### 4. Recommended Range Highlighting
- On the slider track, overlay a colored bar (green/yellow semi-transparent) between `RecommendedMin` and `RecommendedMax`
- The rest of the track stays the default dark color
- If `RecommendedMin`/`RecommendedMax` are not set, no highlight shown

### 5. Mouse Wheel Fine-Tuning
- Mouse wheel over slider: adjust by 1 × TickFrequency
- Ctrl + Mouse wheel: adjust by 0.1 × TickFrequency (fine)
- Shift + Mouse wheel: adjust by 10 × TickFrequency (coarse)
- Only when slider area or control is focused/hovered

### 6. Logarithmic Scale
- When `IsLogarithmic="True"`, the slider maps linearly in log-space
- Internally: `sliderPosition = log(value) / log(max)` and inverse
- Applied to: FxScale, FyScale, MzScale, ForceSensitivity, SteeringLockDegrees, ForceScale, OutputGain
- Value display and text input still show the real (linear) value

### 7. Section Color Coding
- Each section group gets a `SectionColor` brush passed to all its `LabeledSlider` instances
- Colors: 
  - Tyre Forces: `#FF90CAF9` (light blue)
  - Tyre Slip: `#FF4DB6AC` (teal)
  - Tire Grip Feel: `#FF66BB6A` (green)
  - Damping: `#FFFFD600` (yellow)
  - Dynamic Effects: `#FFFF9800` (orange)
  - Output: `#FFE67E22` (the existing accent)
  - Vibration: `#FFCE93D8` (purple)
  - LFE: `#FFEF5350` (red)
  - Signal Processing: `#FF4FC3F7` (cyan)
- Color applied to: slider track fill, section header, thumb glow, value text

### 8. Mini Undo
- Each `LabeledSlider` stores `_previousValue` 
- On Value change (from slider drag or text input), save the old value before applying
- Ctrl+Z when the control is focused: restore `_previousValue`
- Context menu also gets "Undo last change" option (dimmed if no undo available)
- Only one level deep (keeps it simple)

## Implementation Steps

### Step 1: Create `LabeledSlider` UserControl
- New file: `src/AcEvoFfbTuner/Controls/LabeledSlider.xaml` + `.xaml.cs`
- Define all dependency properties
- Build the visual template with all features
- Implement mouse wheel handler, context menu, edit mode logic, undo stack

### Step 2: Define section colors as static resources
- Add section color brushes to `App.xaml` or `MainWindow.xaml` resources

### Step 3: Define recommended ranges
- Add a constants/defaults class (e.g., `SliderDefaults.cs`) that maps each parameter name to its recommended min/max and default value
- These can be driven from the profile's defaults

### Step 4: Replace sliders in MainWindow.xaml
- Replace each `StackPanel > DockPanel > Slider` block with a single `<controls:LabeledSlider>` tag
- Bind all properties
- Do this section by section to avoid breaking things
- ~40 replacements total

### Step 5: Build and verify
- `dotnet build AcEvoFfbTuner.slnx -c Release`
- Visual verification of all sections

## Risk / Considerations
- The WPF default `Slider` template is already overridden globally in `App.xaml`. The `LabeledSlider` will contain its own `Slider` with a private template, so it won't conflict.
- `IsSnapToTickEnabled` with log scale: the log mapping handles this internally; we don't use `IsSnapToTickEnabled` on the underlying slider, we snap ourselves.
- Two-way binding on `Value` must preserve ViewModel `PropertyChanged` notifications — use a standard `DependencyProperty` with `BindsTwoWayByDefault = true`.
