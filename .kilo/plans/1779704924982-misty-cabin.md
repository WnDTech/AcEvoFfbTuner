# Plan: Integrate Assetto Corsa Shared Memory

## Context
Add original Assetto Corsa (AC1) support as a third game source, following the same adapter pattern used for RaceRoom. AC1 uses different MMF names (`Local\acpmf_*`) than AC EVO (`Local\acevo_pmf_*`), and has a simpler graphics/static struct. The physics struct is nearly identical between AC1 and EVO (AC1 lacks the 4 vibration fields at the end). Pipeline isolation is critical — AC1 changes must not affect EVO or R3E.

## Key Insight: AC1 Physics ≈ EVO Physics
AC1's physics struct has the **exact same field layout** as `SPageFilePhysicsEvo` except the 4 vibration fields at the end (KerbVibration, SlipVibrations, RoadVibrations, AbsVibrations). AC1 provides native per-wheel Mz/Fx/Fy/slip data, so the base `FfbPipeline` channel mixer works without modification. The adapter only needs to synthesize vibrations from suspension/tire data (like R3E does).

## Architecture
```
AC1 Shared Memory → AssettoCorsaSharedMemoryReader (adapter) → SPageFilePhysicsEvo/SPageFileGraphicEvo/SPageFileStaticEvo
  → FfbPipeline.Process() (base class, no game-specific subclass needed initially)
  → FFB device output
```

## Files to Create

### 1. `src/AcEvoFfbTuner.Core/SharedMemory/Structs/AcSharedStruct.cs`
AC1-specific structs for graphics and static data. The physics struct reuses `SPageFilePhysicsEvo`.

**AC1 Graphics (`SPageFileGraphicAC`)** — Pack=1 sequential struct matching AC1's memory layout:
- `PacketId` (int)
- `Status` (int — 0=off, 1=replay, 2=live, 3=pause)
- `Velocity` (float[3]) — car world velocity
- `Heading`, `Pitch`, `Roll` (float) — orientation
- `IgnitionOn`, `HandbrakeOn`, `GamePaused` (int)
- `SpeedKmh` (float)
- `IsAbsEnabled`, `IsAbsInAction`, `IsTcInAction`, `IsTcEnabled` (int)
- `IsLightsEnabled`, `IsLightsToggled` (int)
- `AntiStallsEnabled`, `DisableSpeedWarning` (int)
- `VisualSettingsRevision` (int)
- `Rpm` (float)
- `Gear` (int)
- `Gas`, `Clutch` (float)
- `CarLocation` (int — 0=none, 1=track, 2=pitlane, 3=pitbox)
- `SteerAngle` (float — normalized -1..+1)
- `WorldPosition` (float[3])
- `NormalizedPosition` (float)
- `RacePosition` (int)
- `CompletedLaps` (int)
- `CurrentLapInvalidCount` (int)
- `TrackLengthM` (float)
- `DeltaLapTimeMs` (int)
- `EstimatedLapTimeMs` (int)
- `SplitTimeMs` (int)
- `LastSplitTime` (int)
- `CurrentLap` (int)
- `CurrentLapTimeMs` (int)
- `LastLapTimeMs` (int)
- `TimeLeftMs` (int)
- `Position` (int)
- `TrackConfig` (byte[256]) — null-terminated
- `CarModel` (byte[64]) — null-terminated
- `PlayerNickname` (byte[64]) — null-terminated

**AC1 Static (`SPageFileStaticAC`)** — Pack=1 sequential struct:
- `SmVersion` (byte[15])
- `AcVersion` (byte[15])
- `TrackName` (byte[256]) — null-terminated
- `TrackConfiguration` (byte[256]) — null-terminated
- `TrackLengthM` (float)

**NOTE**: AC1's struct layout varies by CSP version. We'll use byte-offset reading (like `StaticFieldReader`) as a fallback for fields that don't align via Marshal.

### 2. `src/AcEvoFfbTuner.Core/SharedMemory/AssettoCorsaSharedMemoryReader.cs`
Adapter following the R3E pattern — reads AC1 MMFs, maps to EVO structs.

- **MMF names**: `Local\acpmf_physics`, `Local\acpmf_graphics`, `Local\acpmf_static`
- **Connection**: Try `MemoryMappedFile.OpenExisting()` on each MMF
- **PacketId dedup**: Track `_lastPhysicsPacketId` / `_lastGraphicsPacketId`
- **`TryReadPhysics()`**: Read AC1 physics buffer → marshal → copy to `SPageFilePhysicsEvo`. The struct fields align 1:1 except vibrations (set to 0, synthesized separately)
- **`TryReadGraphics()`**: Read AC1 graphics buffer → map fields to `SPageFileGraphicEvo`:
  - `Status` → `AcEvoStatus` enum mapping
  - `Rpm`/`Gas`/`Brake` → compute `RpmPercent`, `GasPercent`, `BrakePercent`
  - `CarLocation` → `AcEvoCarLocation` mapping
  - `SteerDegrees` → default 900 (AC1 doesn't expose this directly)
  - `FfbStrength`/`CarFfbMultiplier` → default 1.0 (AC1 doesn't have these in graphics)
  - `CarModel`/`DriverName` → from AC1's string fields
  - `CarCoordinates[0]` → from `WorldPosition`
  - `Npos` → from `NormalizedPosition`
  - `SessionState.CurrentLap` → from `CurrentLap`
- **`TryReadStatic()`**: Read AC1 static buffer → map to `SPageFileStaticEvo`
- **Vibration synthesis** (from suspension/tire data, same technique as R3E):
  - `KerbVibration` — from suspension deflection deltas
  - `SlipVibrations` — from front tire slip ratio
  - `RoadVibrations` — from tire speed deltas
  - `AbsVibrations` — from ABS state + tire speed asymmetry
- **Properties**: `SteeringCenterTrimDeg` (like R3E), `TireGrip` (if available)

### 3. `src/AcEvoFfbTuner.Core/FfbProcessing/AcFfbPipeline.cs`
AC-specific pipeline subclass. Initially minimal — mostly inherits base `FfbPipeline` behavior. May add:
- Gear shift filter override (AC1's FFB can spike on shifts)
- DC blocker on detail path (similar to R3E, for center-bias during cornering)

This is intentionally minimal for v1. The base pipeline handles AC1 well since it provides native per-wheel Mz/Fx/Fy.

## Pipeline Isolation Verification
- `AcFfbPipeline` only processes AC1 data — never touches R3E or EVO pipeline state
- `AssettoCorsaSharedMemoryReader` only reads `acpmf_*` MMFs — never opens `acevo_*` or `$R3E`
- `FfbProfile.ApplyToPipeline()` uses `is AcFfbPipeline` guard — AC settings only apply when AC pipeline is active
- Base `FfbPipeline` has zero AC-specific code — virtual hooks default to no-op
