# Plan: Auto Wheelbase FFB Configuration (Hybrid)

## Problem
Users don't understand the ~50 configurable FFB parameters. Static "Safe - *" presets don't cover all wheelbases and don't adapt to specific hardware characteristics. Users experience severe oscillation because defaults are wrong for their torque level.

## Solution: Hybrid Auto-Config
1. **Smart Wizard** - One-click auto-configuration that detects the wheel and generates a complete custom baseline profile tuned for that specific torque/hardware
2. **Live Auto-Tune** - Optional background mode that monitors driving and makes small incremental corrections to prevent oscillation

### WheelCharacteristics enum
- `DirectDrive` - DD wheels (Moza R5/R9/R12/R16/R21, Fanatec DD, Simagic, Simucube)
- `GearDriven` - Logitech G29/G920
- `BeltDriven` - Thrustmaster T300/TX/T150

### Physics-Based Parameter Scaling (from torque)

**OutputGain**: Inverse power curve. Target ~0.4-0.5Nm peak at wheel.
```
OutputGain = clamp(2.5 / sqrt(torqueNm), 0.25, 1.0)
```
- 2Nm â†’ 1.0, 5.5Nm â†’ 0.73, 9Nm â†’ 0.56, 15Nm â†’ 0.45, 25Nm â†’ 0.35

**NormalizationScale**: Higher torque needs higher sensitivity.
```
NormScale = 250 + (torqueNm - 2) * 22
```
- 2Nm â†’ 250, 5.5Nm â†’ 327, 9Nm â†’ 404, 15Nm â†’ 536, 25Nm â†’ 756

**CompressionPower**: Higher torque compresses more to prevent spikes.
```
CompPower = 1.2 + torqueNm * 0.02
```
- 2Nm â†’ 1.24, 5.5Nm â†’ 1.31, 9Nm â†’ 1.38, 15Nm â†’ 1.50, 25Nm â†’ 1.70

**MaxSlewRate**: DD wheels are faster, need tighter slew limiting.
```
slewRate = isDirectDrive ? clamp(0.15 - torqueNm * 0.003, 0.05, 0.15) : 0.20
```

**Damping**: Scales with torque. DD wheels need more damping to prevent oscillation.
```
speedDamp = isDirectDrive ? clamp(0.3 + torqueNm * 0.05, 0.5, 1.5) : clamp(0.2 + torqueNm * 0.1, 0.2, 0.5)
friction = isDirectDrive ? clamp(0.2 + torqueNm * 0.02, 0.3, 0.8) : clamp(0.15 + torqueNm * 0.05, 0.15, 0.4)
inertia = isDirectDrive ? clamp(0.1 + torqueNm * 0.01, 0.15, 0.4) : clamp(0.05 + torqueNm * 0.02, 0.05, 0.15)
```

**CenterSuppressionDegrees**: DD needs wider dead zone.
```
centerSupp = isDirectDrive ? 6.0 : 4.0
```

**CenterBlendDegrees**: DD needs wider Fy suppression.
```
blendDeg = isDirectDrive ? 2.0 : 1.0
```

**HysteresisThreshold**: DD more sensitive to micro-oscillation.
```
hystThresh = isDirectDrive ? 0.02 : 0.01
```

**NoiseFloor**: DD picks up more noise.
```
noiseFloor = isDirectDrive ? 0.008 : 0.004
```

**LowSpeedDampingBoost**: Higher for DD to prevent hunting.
```
boost = isDirectDrive ? 3.0 : 2.0
```

**Vibration Gains**: Inversely proportional to torque (stronger wheels need less vibration gain).
```
vibMaster = clamp(0.6 - torqueNm * 0.015, 0.15, 0.6)
suspensionRoadGain = clamp(1.2 - torqueNm * 0.03, 0.3, 1.2)
```

**Channel Gains** (always set, regardless of hardware):
- MzFront: 1.0 (primary channel)
- FxFront: 0.12
- FyFront: 0.15

All other parameters get sensible defaults matching the existing `CreateSafeWheelbaseProfile()` pattern.

### EvoDetectedSettings Integration
When game telemetry is available, use `EvoSettingsDetector` values for OutputGain, NormalizationScale, and SteeringLockDegrees (override the torque-based estimates with actual in-game data).

### Live Tuning Rules
Only adjusts **oscillation-critical** parameters, never changes feel/taste parameters:

1. **Oscillation detected** â†’ Reduce MaxSlewRate by 5% (floor: 0.03)
2. **Suspicious snap on straight (road vibration)** â†’ Reduce SuspensionRoadGain by 8% (floor: 0.2)
3. **Clipping > 20%** â†’ Reduce OutputGain by 3% (floor: 0.2)
4. **High-speed snap (avg > 150km/h)** â†’ Reduce MaxSlewRate additional 3%
5. **Bounce-back after hysteresis release** â†’ Increase HysteresisThreshold by 0.002 (ceiling: 0.04)

### Safeguards
- Maximum 1 correction per 10 seconds (prevents rapid oscillation of corrections)
- Each parameter has hard floors/ceilings
- Corrections are cumulative but bounded
- `CorrectionLog` tracks every change for transparency
- Can be reset to restore the wizard baseline

### `AutoSetup()` Flow
1. Check device connected â†’ get `DeviceName`, `WheelMaxTorqueNm`
2. Detect `WheelCharacteristics` from device name
3. Try `EvoSettingsDetector.DetectFromRaw()` if game connected (for in-game FFB strength/car multiplier)
4. Call `WheelbaseAutoConfigurator.GenerateProfile(torque, deviceName, evoSettings)`
5. Set profile name to `"Auto - {DeviceName} ({torque}Nm)"`
6. Add to `Profiles` collection
7. Set as `SelectedProfile` (triggers `ApplyToPipeline` + `LoadProfileValues`)
8. Save profile via `_profileManager`
9. Configure `LiveAutoTuner` if enabled
10. Update status: `"Auto-configured for {device} â€” {torque}Nm, {type} wheelbase"`

### LiveAutoTune Integration
- In `OnUiUpdate()`, after diagnostic summary is available, call `_liveAutoTuner.OnLapSummary(summary)`
- `OnLiveAutoTuneEnabledChanged` â†’ enable/disable the tuner
- When tuner makes a correction, update the corresponding ViewModel property (so sliders update in UI)
- Log corrections to `LastCorrectionText`
