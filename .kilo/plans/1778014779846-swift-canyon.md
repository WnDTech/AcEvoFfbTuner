# UI Redesign Plan — AC Evo FFB Tuner

## Problem Statement

The current UI has 10+ tabs, 2350+ lines of XAML, no clear visual hierarchy, and crams too much into the header/status bars. Newcomers are overwhelmed by walls of sliders; pros have to hunt through tabs for what they need.

## Design Principles

1. **Task-oriented** — Organize around what the user is doing (tuning, analyzing, configuring)
2. **Progressive disclosure** — Essentials visible, details on demand
3. **Sidebar navigation** — Always-visible, scalable, more content area
4. **Refined dark theme** — Same character, better contrast, less noise

## 3. FFB Tuning (Main Workspace)

This is the biggest change. Instead of two massive columns of sliders, use a **collapsible section accordion** with clear visual grouping and progressive disclosure.

### Layout: Three Zones

```

│  Header: FFB Tuning                    [Output: ON] │
│  Subtitle: BMW GT3 · Moza R5 · Replace Mode        │
├─────────────────────────────────────────────────────┤
│                                                     │
│  ┌─ OUTPUT (always visible) ─────────────────────┐  │
│  │ Output Gain [====●======] 1.0   (~5.5 Nm)    │  │
│  │ Force Limit [======●====] 0.80  (~4.4 Nm)    │  │
│  │ Max Slew   [===●=======] 0.40                 │  │
│  │ [▼ Advanced Output Settings]                   │  │
│  └───────────────────────────────────────────────┘  │
│                                                     │
│  ┌─ TYRE FORCES ──────────────────── [▼ Expand] ─┐  │
│  │ Mz Front  [====●======] 1.000  ☑ Enabled      │  │
│  │ Fx Front  [●=========] 0.100  ☑ Enabled      │  │
│  │ Fy Front  [●=========] 0.100  ☑ Enabled      │  │
│  │ [▼ Normalization Scales] [▼ Rear Axle]         │  │
│  └───────────────────────────────────────────────┘  │
│                                                     │
│  ┌─ DAMPING & FRICTION ──────────── [▼ Expand] ─┐  │
│  │ Speed Damp [===●======] 0.300                  │  │
│  │ Friction   [==●=======] 0.250                  │  │
│  │ [▼ Detailed Settings]                          │  │
│  └───────────────────────────────────────────────┘  │
│                                                     │
│  ┌─ DYNAMIC EFFECTS ─────────────── [▼ Expand] ─┐  │
│  └───────────────────────────────────────────────┘  │
│                                                     │
│  ┌─ TYRE SLIP ──────────────────── [▼ Expand] ─┐  │
│  └───────────────────────────────────────────────┘  │

```

### Section Design Rules
- **OUTPUT** is always visible at the top (most frequently adjusted)
- Each section has a colored left border matching the existing SectionColor scheme
- Basic controls (2-3 sliders) visible when expanded; "Detailed Settings" sub-expander for advanced
- Section headers show key value summary when collapsed (e.g., "TYRE FORCES — Mz: 1.0, Fx: 0.1, Fy: 0.1")
- Expander state remembered per session

### Collapsed sections show a summary line
```
▸ TYRE SLIP  Ratio: 0.00 · Angle: 0.00 · Threshold: 0.05
```

## 7. Implementation Plan

### Phase 0: Icon Preview (before anything else)
1. **Create `Views/IconPreviewWindow.xaml`** — A standalone test window showing all 7 sidebar icons rendered at actual size (28x28 in a Viewbox) with their labels
2. Each icon shown in two states: default (dim) and selected (accent orange) on the dark sidebar background
3. User reviews, approves, or requests changes to individual icons before any structural work begins
4. This window is temporary — removed before release

### Phase 1: Foundation (structurally safe changes)
1. **Create `SidebarControl.xaml`** — Custom sidebar UserControl with navigation items
2. **Create `NavigationView.cs`** — ViewModel for managing active page
3. **Create page UserControls** — Extract each tab into its own `UserControl`:
   - `Views/Pages/HomePage.xaml`
   - `Views/Pages/FfbTuningPage.xaml`  
   - `Views/Pages/EqualizerPage.xaml`
   - `Views/Pages/TrackMapPage.xaml`
   - `Views/Pages/TelemetryPage.xaml`
   - `Views/Pages/DevicesPage.xaml`
   - `Views/Pages/ProfilesPage.xaml`
4. **Update `MainWindow.xaml`** — Replace TabControl with sidebar + ContentPresenter
5. **Update `App.xaml`** — Add refined color palette as resource dictionary

### Phase 2: Home Dashboard
6. **Build Home page** — Status cards, start/stop, live telemetry mini, quick-start guide
7. **Move connection/start logic** — From header to Home page
8. **Add quick-start guide card** — Condensed from current Testing Guide tab

### Phase 3: FFB Tuning Rework
9. **Build sectioned accordion** — OUTPUT always visible, other sections collapsible
10. **Add section summary in collapsed state** — Show key values when collapsed
11. **Add colored left borders** — Section colors on cards
12. **Reorganize section grouping** — Move Output to top, group logically

### Phase 4: Header & Status Bar
13. **Simplify header** — Connection pill, profile pill, start/panic only
14. **Create device popover** — Click connection pill to see device list
15. **Create profile popover** — Click profile pill to see profile list + save
16. **Simplify status bar** — Essential info only

### Phase 5: Visual Polish
17. **Apply refined color palette** — Update all hardcoded colors
18. **Improve LabeledSlider** — Better spacing, section-colored track fill
19. **Add hover effects** — Cards, buttons, sidebar items
20. **Typography cleanup** — Consistent sizing, better hierarchy

## 8. Technical Considerations

- **MVVM**: Each page gets its own ViewModel, injected with shared services from `MainViewModel`
- **No framework change**: Still WPF, still XAML, still CommunityToolkit.Mvvm
- **Backward compatible**: All existing bindings and commands preserved, just reorganized
- **Performance**: Pages are lazy-loaded (created on first navigation, then cached)
- **Sidebar collapse**: Toggle button to collapse sidebar to icon-only (60px) for more content space on smaller screens
